<objective>
## Goal
Harden `text_processing.split_text` with three concrete improvements: (1) fix the position-accuracy bug where `text.find(paragraph_or_sentence, cursor)` can return the wrong offset when the same substring appears more than once in the input; (2) introduce a module-level character budget aligned with the OpenAI TTS 4096-char input limit and assert chunks never exceed it; (3) lock the position-metadata contract via a reconstruction-style test. Add edge-case coverage for unicode, duplicate paragraphs, empty-paragraph interspersing, and exact-boundary lengths.

## Purpose
Pre-PAUL commit `f2ff098` already shipped paragraph/sentence-aware chunking, so the structural work is done. What's missing for Phase 3 is the **integrity layer**: the position metadata is what Phase 4 (GUI progress bar) and any future re-sync feature depend on, and right now `text.find(...)` can quietly return a wrong offset when the input has repeated phrases (very common in real prose: "He said." "He said." or chapter-title repetition). The 3500-char default is arbitrary; OpenAI's hard ceiling is 4096 and Phase 6.2 Kokoro likely has its own. Naming the constant + asserting it in tests prevents a future contributor from raising it past the API limit and producing 400s in production.

The 179-test regression net stays green; the position-accuracy fix is a silent improvement (no characterization needed — wrong positions were a bug, not a documented behavior).

## Output
- `text_processing.py` updates:
  - Module-level constants: `OPENAI_TTS_MAX_INPUT_CHARS = 4096` (hard ceiling per OpenAI docs), `DEFAULT_CHUNK_MAX = 3500` (safe margin under the ceiling, used as `split_text` default)
  - `split_text` default parameter `max_length` references `DEFAULT_CHUNK_MAX` instead of the inline `3500`
  - Position-accuracy fix: replace `text.find(paragraph, cursor)` with a forward-only cursor pattern that finds the NEXT occurrence at-or-after the cursor and advances the cursor PAST the matched substring (so repeated phrases get distinct positions in source order). Same fix for sentence-level `text.find(sentence, ...)` and hard-split `text.find(piece, ...)`.

    3. **Fix the position-accuracy bug.** Replace the body of `split_text` with the following forward-cursor pattern. The key change: maintain ONE `find_cursor` that monotonically advances past every matched substring. Every `text.find` call uses `find_cursor` as `start`, and on success advances `find_cursor` PAST the matched substring before the next call. This guarantees repeated substrings get distinct, source-ordered positions.

       Replace the body of `split_text` (after the `text = _normalize_text(text)` line) with:
       ```python
           chunks = []
           positions = []
           sentences = []
           if not text:
               return chunks, positions, sentences

           raw_paragraphs = [paragraph.strip() for paragraph in re.split(r'\n\s*\n', text) if paragraph.strip()]

           # Forward-only cursor: every subsequent find searches strictly AT-OR-AFTER this.
           # Each successful match advances the cursor past the matched substring so duplicates
           # in the source resolve to distinct, source-ordered offsets.
           find_cursor = 0
           current_chunk = ""
           current_position = 0

           def _locate(substring, fallback):
               nonlocal find_cursor
               idx = text.find(substring, find_cursor) if substring else -1
               if idx == -1:
                   # Audit-added S2: defensive observability. paragraphs/sentences are
                   # extracted from the normalized text via re.split, so find() SHOULD
                   # always succeed. If it doesn't, normalization has desynced from search
                   # — surface it under DEBUG so maintainers see it in CI runs that opt in.
                   logging.debug(
                       "text_processing._locate: substring not found at-or-after cursor=%d; "
                       "using fallback. substring_preview=%r",
                       find_cursor, substring[:30],
                   )
                   return fallback
               find_cursor = idx + len(substring)
               return idx

           def flush_chunk():
               nonlocal current_chunk, current_position
               cleaned = current_chunk.strip()
               if cleaned:
                   chunks.append(cleaned)
                   positions.append(current_position)
                   sentences.append(_chunk_preview(cleaned))
               current_chunk = ""

           for paragraph in raw_paragraphs:
               paragraph_start = _locate(paragraph, find_cursor)

               if len(paragraph) > max_length:
                   for sentence in _sentence_split(paragraph):
                       sentence_start = _locate(sentence, paragraph_start)
                       candidate = f"{current_chunk} {sentence}".strip() if current_chunk else sentence
                       if not current_chunk:
                           current_position = sentence_start

                       if len(candidate) <= max_length:
                           current_chunk = candidate
                           continue

                       flush_chunk()

                       if len(sentence) <= max_length:
                           current_chunk = sentence
                           current_position = sentence_start
                           continue

                       hard_start = 0
                       while hard_start < len(sentence):
                           hard_end = min(hard_start + max_length, len(sentence))
                           piece = sentence[hard_start:hard_end].strip()
                           if piece:
                               # Hard-split pieces use sentence_start + hard_start directly;
                               # no need to find() since we know the slice location exactly.
                               piece_pos = sentence_start + hard_start
                               chunks.append(piece)
                               positions.append(piece_pos)
                               sentences.append(_chunk_preview(piece))
                           hard_start = hard_end
                   continue

               candidate = f"{current_chunk}\n\n{paragraph}".strip() if current_chunk else paragraph.strip()
               if not current_chunk:
                   current_position = paragraph_start

               if len(candidate) <= max_length:
                   current_chunk = candidate
               else:
                   flush_chunk()
                   current_chunk = paragraph.strip()
                   current_position = paragraph_start

           flush_chunk()
           return chunks, positions, sentences
       ```

       Key correctness properties this delivers:
       - `find_cursor` only advances; no backtracking → repeated substrings get distinct offsets in source order
       - Hard-split pieces compute position arithmetically from `sentence_start + hard_start` (exact, no find() needed)
       - The fallback path when `find` returns -1 uses the current `find_cursor` value (forward-safe) — no silent backwards jump

       class TestSplitTextPositions:
           def _reconstruct_check(self, text, chunks, positions):
               """For each chunk, the text starting at positions[i] must begin with the
               chunk's content (ignoring leading whitespace from chunk strip + paragraph
               normalization). We check the first N word-chars to keep the test robust to
               whitespace-only normalization differences.

               Audit-added S4: also assert non-empty chunks and a length-floor on the
               source slice so a past-EOF position fails fast instead of slipping through
               on a coincidental short prefix match.
               """
               for chunk, pos in zip(chunks, positions):
                   # S4: no empty chunks should ever land in the list
                   assert len(chunk) > 0, f"empty chunk at position {pos}"
                   prefix = chunk.lstrip()[:40]
                   # S4: source slice must have enough characters to compare meaningfully
                   source_slice = text[pos:pos + len(prefix) + 10].lstrip()
                   assert len(source_slice) >= min(len(prefix), 10), (
                       f"source slice too short at position {pos} "
                       f"(text length={len(text)}, slice={source_slice!r})"
                   )
                   assert source_slice.startswith(prefix[:20]), (
                       f"Reconstruction failed: chunk prefix {prefix[:20]!r} "
                       f"not found at position {pos} (source slice: {source_slice[:30]!r})"
                   )

           def test_positions_are_monotonic_non_decreasing(self):
               text = "Alpha sentence.\n\nBeta sentence.\n\nGamma sentence."
               chunks, positions, _ = split_text(text, max_length=20)
               assert positions == sorted(positions), (
                   f"positions not monotonic: {positions}"
               )

           def test_distinct_chunks_have_distinct_positions(self):
               """Audit S3: trap the [0, 0, 0] regression — a broken implementation that
               always returns 0 would pass the monotonic test. This asserts that when
               distinct-content chunks exist, at least 2 positions differ."""
               text = "Alpha sentence.\n\nBeta sentence.\n\nGamma sentence."
               chunks, positions, _ = split_text(text, max_length=20)
               assert len(chunks) >= 2, f"expected ≥ 2 chunks, got {len(chunks)}"
               assert len(set(positions)) >= 2, (
                   f"all positions collapsed to the same value: {positions}"
               )

           def test_duplicate_paragraphs_get_distinct_positions(self):
               # Regression for the position-accuracy bug: two identical paragraphs must
               # resolve to two distinct, source-ordered offsets — not one offset reused.
               text = "He said.\n\nShe said.\n\nHe said."
               chunks, positions, _ = split_text(text, max_length=10)
               # All three paragraphs are short, so each becomes its own chunk (or pair fits).
               # The two "He said." occurrences MUST have distinct positions.
               he_positions = [p for c, p in zip(chunks, positions) if "He said" in c]
               assert len(he_positions) == 2, f"expected 2 'He said' chunks, got {len(he_positions)}: chunks={chunks}"
               assert he_positions[0] != he_positions[1], (
                   f"duplicate 'He said' chunks collapsed to same position: {he_positions}"
               )
               assert he_positions[0] < he_positions[1]

           def test_reconstruction_short_input(self):
               text = "Hello world. This is short."
               chunks, positions, _ = split_text(text)
               self._reconstruct_check(text, chunks, positions)

           def test_reconstruction_multi_paragraph(self):
               text = "First paragraph here.\n\nSecond paragraph here.\n\nThird paragraph here."
               chunks, positions, _ = split_text(text, max_length=20)
               self._reconstruct_check(text, chunks, positions)

           def test_reconstruction_hard_split_sentence(self):
               # Single long sentence that must hard-split.
               text = "a" * 500

               # Each hard-split piece's position should reconstruct to "aaaa..."
               for chunk, pos in zip(chunks, positions):
                   assert text[pos:pos + len(chunk)] == chunk
