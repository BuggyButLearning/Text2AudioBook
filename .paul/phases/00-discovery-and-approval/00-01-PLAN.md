<objective>
## Goal
Obtain explicit stakeholder approval of MODERNIZATION_PRD.md and resolve every open question in §14 (final planning assumptions) and §14.2 (HuggingFace-provider open questions), so that Phase 1 (Architecture and Configuration) can start with no ambiguity.

## Purpose
Phase 0 is the gatekeeper: no code changes happen until the PRD is signed off and the scope of optional providers (Kokoro / VibeVoice) is locked. Skipping this would risk building a feature (e.g. VibeVoice GPU path) the stakeholder may decline once costs/limits are surfaced. This phase also locks the answers needed to write deterministic plans in Phase 1+.

## Output
1. `.paul/phases/00-discovery-and-approval/00-01-APPROVAL-PACKET.md` — single-page packet summarising the PRD for review + every open question with recommended default.
2. Resolution of each open question, recorded inline in the APPROVAL-PACKET and propagated into:
   - PRD `§14.1` / `§14.2` (status updated from "Open" → resolution + date)
   - `.paul/PROJECT.md` `Key Decisions` table (new rows for each resolved question)
3. Explicit stakeholder approval signal ("approved") captured in the SUMMARY.
4. SUMMARY.md at `.paul/phases/00-discovery-and-approval/00-01-SUMMARY.md`.
</objective>
