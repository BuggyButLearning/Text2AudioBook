<objective>
## Goal
Establish a baseline characterization test suite that locks the current behavior of all pure-logic helpers in `settings.py`, `text_processing.py`, `tts_conversion.py`, and `combine_and_convert.py` — so that the Phase 1+ refactor lands against a regression net instead of bare metal.

## Purpose
PRD §10 mandates testing; PRD §4.2.F flags "Test coverage is absent" as a key problem. The cheapest, lowest-risk first move is characterization tests against today's code — they don't change any source, don't cost API money, don't require human judgement, and they give Phase 1 (config refactor) and Phase 3 (chunking refactor) an immediate red-flag signal when behavior shifts unintentionally. This plan runs in parallel with the 00-01 approval checkpoint — does not block PRD sign-off.

## Output
- New `tests/` package with conftest + 4 test modules
- All tests pass against current `main` source (snapshot of current behavior)
- `pytest` runs green via `conda run --name text2audiobook python -m pytest tests`
- SUMMARY.md at `.paul/phases/00-discovery-and-approval/00-02-SUMMARY.md` listing what was locked
</objective>

### Mock patch paths (audit-added M2):
Because `tts_conversion.list_openai_models` performs a DEFERRED import

`tts_conversion.OpenAI` (which does not exist as a module attribute until
after the deferred import runs). Tests MUST use:
  `monkeypatch.setattr("openai.OpenAI", FakeOpenAI)`

`tests/test_tts_conversion.py` so Phase 1's refactor (which may hoist
the import to module top-level) can flip the patch path knowingly.
