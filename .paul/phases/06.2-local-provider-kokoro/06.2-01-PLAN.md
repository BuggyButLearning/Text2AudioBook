<objective>
## Goal
Land the Kokoro-82M local synthesis path. New `kokoro_synthesis.py` module owns lazy import of the `kokoro` package, `espeak-ng` system-dep probe, and `_write_kokoro_speech(chunk, file_path, model, voice, ...)` synthesis helper. `tts_conversion.convert_text_chunk_to_speech` dispatches `Ollama → unsupported`, `Kokoro → _write_kokoro_speech`, `OpenAI → _write_openai_speech` (current behavior). Each Kokoro chunk synthesizes to WAV then converts to MP3 via pydub so concatenation stays MP3-uniform.

## Purpose
- v0.1's recommended local default (PRD §FR-1A; Phase 0 §14.2(6)).
- Apache 2.0 license; CPU-capable (no GPU required).
- ~500 MB HF download to `~/.cache/huggingface`; pinned model revision already in `providers.PROVIDER_REGISTRY["Kokoro"]`.
- Requires `espeak-ng` system dep (Windows: .msi installer; Linux: apt).

## Output
- New `kokoro_synthesis.py`:
  - `kokoro_available() -> tuple[bool, str | None]` -- import probe (True if `import kokoro` works)
  - `espeak_ng_available(runner=None) -> tuple[bool, str | None]` -- subprocess probe of `espeak-ng --version` with injection seam
  - `kokoro_ready() -> tuple[bool, str | None]` -- composes both probes
  - `_write_kokoro_speech(chunk, wav_path, model, voice, lang_code="a", speed=1.0)` -- runs `kokoro.KPipeline(lang_code=...).generate(chunk, voice=voice, speed=speed)` and writes WAV via soundfile
  - `_convert_wav_to_mp3(wav_path, mp3_path)` -- pydub helper
- `tts_conversion.convert_text_chunk_to_speech`: Kokoro branch synthesizes WAV then converts to MP3, returning MP3 path so concatenation stays uniform.
- `main.start_conversion`: replace blanket "Phase 6.2 pending" guard with `kokoro_ready()` check; show actionable error if kokoro lib missing OR espeak-ng missing.
- Deps added to requirements.txt + environment.yml: `kokoro>=0.9.4`, `soundfile>=0.12.1`, `huggingface_hub>=0.20.0`.
- Tests use `monkeypatch` on the lazy-imported module so they work even when `kokoro` is not installed.
</objective>
