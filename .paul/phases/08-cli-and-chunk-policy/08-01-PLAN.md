## Goal
Two new features, fully documented for AI consumption:

1. **CLI interface** — `python -m text2audiobook <command>` (also `python cli.py`). Every GUI setting overridable via flags. `--json` machine-readable output. Stable exit codes. Discovery commands (`list-providers`, `list-models`, `list-voices`). Self-documenting `--help` per subcommand.

2. **Per-(provider, model) chunk_max** — research-backed defaults (OpenAI tts-1 = 3500, Kokoro = 2000, Ollama = 1000) consumed by both GUI and CLI. User can override per provider/model via `config.json` `chunk_overrides` map OR CLI `--chunk-max N`.

## Purpose
- Today the only entry point is the GUI; AI agents can't drive synthesis without screen automation.
- `text_processing.DEFAULT_CHUNK_MAX = 3500` is OpenAI-tuned but applied to every provider. Kokoro auto-chunks internally on phoneme count (cap=510), so feeding it 3500-char paragraphs is fine but feeding it 200-char fragments wastes per-chunk model warmup. Ollama-bark-style models choke past ~300 input tokens.
- AI agents need stable contracts: a CLI doc, machine-readable JSON output, predictable exit codes.

## Research-backed chunk_max defaults

| Provider | Model pattern | chunk_max | Source / rationale |
|----------|--------------|-----------|--------------------|
| OpenAI | `tts-*` | 3500 | Hard ceiling 4096 chars per request (PRD §FR-2). 3500 leaves 15% headroom for whitespace normalization. |
| Kokoro | `kokoro*` | 2000 | KPipeline auto-splits internally at 510 phonemes (`kokoro/pipeline.py` waterfall on `!.?…` then `:;` then `,—`). 2000-char app-level chunks reduce per-chunk model warmup overhead while staying well above one paragraph. |
| Ollama | `bark*` / `tts*` / `speech*` | 1000 | bark-style models commonly handle ~300 tokens (~1000 chars) before quality degrades. Conservative; user can override. |

### CLI commands
| Command | Purpose | JSON-friendly |
|---------|---------|---------------|
| `synthesize` | Convert text/markdown → MP3 | Yes |

### synthesize flags
- `--input PATH` (required) — .txt / .md / .markdown
- `--output PATH` (required) — .mp3
- `--provider {OpenAI,Ollama,Kokoro}` (default: from config)
- `--model NAME` (default: provider's first fallback)
- `--voice NAME` (default: provider's first voice)
- `--quality {Balanced,Best Quality,Fast}` (default: Balanced)
- `--chunk-max N` (override resolved policy)
- `--speed FLOAT` (default: from preset)
- `--max-concurrency N` (default: 2, clamped to 1 for local providers)
- `--openai-api-key KEY` (else env / key.txt)
- `--ollama-base-url URL` (else env / default)
- `--json` — emit progress + result as JSON Lines on stdout
- `--quiet` — suppress progress
- `--dry-run` — validate args + resolve config, print plan, don't synthesize

### Exit codes
- 0 success
- 1 invalid arguments / validation failure
- 2 synthesis failure (chunk error / network / etc.)
- 3 provider-not-ready (Kokoro lib missing / Ollama unreachable)
- 4 input file unreadable

### JSON Lines schema (`--json`)
```
{"event": "start", "provider": "...", "model": "...", "chunks": 3, "chunk_max": 3500}
{"event": "chunk", "index": 1, "total": 3, "status": "converted", "elapsed_ms": 1234}
{"event": "complete", "output": "path/to/out.mp3", "duration_ms": 4500}
{"event": "error", "stage": "synthesis", "message": "..."}
```

`chunk-policy --json` →

`resolve_chunk_max(provider, model=None, overrides=None)` returns: 3500 for OpenAI*, 2000 for Kokoro*, 1000 for Ollama*. Falls back to 3500 for unknown providers. Override precedence: explicit overrides[`{provider}:{model}`] > overrides[provider] > default.

## AC-2: split_text accepts chunk_max parameter and respects it
`text_processing.split_text(text, max_length=N)` already accepts max_length. Wire chunk_policy.resolve_chunk_max() into the call site in tts_conversion.convert_text_to_speech (and main.py worker).
