# 02.1-01 SUMMARY — Model Discovery and Selection

## Outcome
Extracted OpenAI / Ollama TTS-model discovery from `tts_conversion.py` into a new `model_discovery.py` module with a frozen `DiscoveryResult` return type, explicit `Source` enum (LIVE / FALLBACK / EMPTY), per-(provider, canonical-identity) cache with explicit `invalidate_cache()`, registry-driven Ollama allowlist filter (PRD §14.1(a) live), and credential-scrubbing on the exception path. Back-compat shims in `tts_conversion.py` preserve `main.py`'s import path. Regression suite: **179 passed in 0.67s, exit 0, zero DeprecationWarning, zero new third-party deps**.

## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Extract discovery into model_discovery.py vs in-place refactor | Discovery has distinct concerns (HTTP, cache, error reporting) from synthesis; Phase 4 GUI should import only the surface it needs | Phase 4 imports only model_discovery; tts_conversion remains synthesis-focused |
| Thin shims in tts_conversion.py preserve `list_*_models` symbols | main.py + existing test imports stay valid; zero risk in a phase that doesn't own GUI | Zero source-of-callers refactor needed |
| FALLBACK + EMPTY cached identically to LIVE | Predictable UX; no surprise refresh storms; "Refresh Models" button is the sole recovery path | Phase 4 wiring: "Refresh" = invalidate + discover; documented + test-enforced |
| `Source.EMPTY` distinguishes "upstream responded but yielded nothing" from `Source.FALLBACK` (exception) | Phase 4 GUI can route messaging correctly (red "down" banner vs yellow "no models" banner) | Audit S1 invariant; Phase 4 inherits a clean semantic surface |
