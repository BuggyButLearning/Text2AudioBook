<objective>
## Goal
Extract OpenAI / Ollama model discovery out of `tts_conversion.py` into a dedicated `model_discovery.py` module. Add a structured `DiscoveryResult` return type (provider, models tuple, source enum {LIVE, FALLBACK, EMPTY}, error message), apply the registry's `model_pattern` as an Ollama curated-allowlist filter (new behavior — currently Ollama returns ALL `/api/tags` entries unfiltered), and add an explicit per-(provider, base_url) cache with a public `invalidate()` for the `Refresh Models` UI flow that Phase 4 will wire up. Preserve back-compat so `main.py` and the existing test suite keep importing `list_available_models` from `tts_conversion` unchanged.

## Purpose
Phase 2 left `list_openai_models` / `list_ollama_models` / `list_available_models` intentionally untouched — Phase 2.1 owns the discovery surface polish. Three motivations:

1. **Single-responsibility:** `tts_conversion.py` is synthesis. Discovery has its own concerns (HTTP, caching, error reporting) and belongs in its own module so Phase 4 (GUI) can import + cache it without dragging in synthesis.
2. **Curated Ollama allowlist:** PRD §14.1(a) says the Ollama path must "validate capabilities, warn/block unsupported models". Today `list_ollama_models` returns everything `/api/tags` reports. Phase 2.1 applies the same `model_pattern` filter that `_validate_ollama_model_support` already uses, so a user with `llama3` installed won't see it in the TTS dropdown.
3. **Refresh Models semantics:** Phase 4 needs a `Refresh Models` button. That requires an explicit cache + invalidation entry point. Without one, every dropdown render hits the network.

The 161-test regression net stays green; the new behavior shift (Ollama filtering) gets characterized with `# CHARACTERIZED — Phase 2.1` comments on tests it changes.

    ```python
    """
    Model-discovery module (Phase 2.1).

    Owns OpenAI / Ollama TTS-model listing. Returns a frozen DiscoveryResult
    with explicit source labeling (LIVE / FALLBACK / EMPTY) so callers (GUI,
    tests, audit log) can distinguish "fresh from the network" from "registry
    fallback because the network failed".

    Discovery is decoupled from synthesis: tts_conversion.py keeps thin shim
    functions that delegate here, so callers like main.py don't need to change
    their imports.

    The Ollama path applies the registry's model_pattern as a curated allowlist
    (PRD §14.1(a)) — Ollama servers commonly report non-TTS models (llama3,
    mistral) that are not synthesis-capable and must not appear in the UI
    dropdown.

    Cache is per-(provider, key) and only invalidates on explicit
    invalidate_cache() — designed for a "Refresh Models" UI button (Phase 4).
    """
    import logging
    import re
    from dataclasses import dataclass
    from enum import Enum
    from typing import Mapping

    import providers


    class Source(str, Enum):
        LIVE = "live"
        FALLBACK = "fallback"
        EMPTY = "empty"


    @dataclass(frozen=True)
    class DiscoveryResult:
        provider: str
        models: tuple[str, ...]
        source: Source
        error: str | None = None


    # Compile registry patterns ONCE at module import (Phase 2 audit S1 pattern).
    _OPENAI_MODEL_RE = re.compile(providers.PROVIDER_REGISTRY["OpenAI"].model_pattern)
    _OLLAMA_MODEL_RE = re.compile(providers.PROVIDER_REGISTRY["Ollama"].model_pattern, re.IGNORECASE)


    # Cache: key = (provider, identity) where identity = api_key for hosted, base_url for local.
    _CACHE: dict[tuple[str, str | None], DiscoveryResult] = {}


    def invalidate_cache(provider: str | None = None) -> None:
        if provider is None:
            _CACHE.clear()
            return
        # Drop all entries for the given provider regardless of identity.
        for key in [k for k in _CACHE if k[0] == provider]:
            _CACHE.pop(key, None)


    def _scrub_api_key(message: str, api_key: str | None) -> str:  # audit-added M1
        """Strip the api_key value from a log/error string before it escapes the function.
        Defensive: openai SDK errors can include URL-encoded request payloads that contain
        the Bearer token in some failure modes."""
        if not api_key or not message:
            return message
        return message.replace(api_key, "***REDACTED***")


    def _discover_openai(api_key: str | None) -> DiscoveryResult:
        cap = providers.PROVIDER_REGISTRY["OpenAI"]
        try:
            from openai import OpenAI
            client = OpenAI(api_key=api_key) if api_key else OpenAI()
            models = client.models.list()
            ids = [m.id for m in models.data]
            filtered = sorted({mid for mid in ids if _OPENAI_MODEL_RE.match(mid)})
            if filtered:
                return DiscoveryResult("OpenAI", tuple(filtered), Source.LIVE, None)
            # Audit-added S1: live-but-allowlist-empty is EMPTY (not FALLBACK).
            # Distinguishes "your OpenAI account has no TTS models" from "OpenAI is down".
            logging.info("OpenAI discovery: API responded OK but no TTS models matched allowlist")
            return DiscoveryResult("OpenAI", (), Source.EMPTY, "no models matched allowlist")
        except Exception as exc:
            scrubbed = _scrub_api_key(str(exc), api_key)  # audit-added M1
            logging.warning("OpenAI discovery failed; using registry fallback: %s", scrubbed)
            return DiscoveryResult("OpenAI", tuple(cap.fallback_models), Source.FALLBACK, scrubbed)


    def _discover_ollama(canonical_url: str) -> DiscoveryResult:
        # Audit-added S2: caller is responsible for passing the canonical URL.
        # discover_models() normalizes once and uses the canonical form for both
        # cache key and this function — no re-normalization here.
        import requests
        url = canonical_url
        try:
            response = requests.get(f"{url}/api/tags", timeout=10)
            response.raise_for_status()
            payload = response.json()
            entries = payload.get("models", []) if isinstance(payload, dict) else []
            names = []
            for entry in entries:
                if isinstance(entry, dict) and isinstance(entry.get("name"), str):
                    names.append(entry["name"])
            total = len(names)
            filtered = sorted({n for n in names if _OLLAMA_MODEL_RE.search(n)})
            logging.info("Ollama discovery: %d models reported, %d after registry filter", total, len(filtered))
            if filtered:
                return DiscoveryResult("Ollama", tuple(filtered), Source.LIVE, None)
            if total > 0:
                # Server responded but nothing matched the allowlist.
                return DiscoveryResult("Ollama", (), Source.EMPTY, "no models matched registry allowlist")
            return DiscoveryResult("Ollama", (), Source.EMPTY, None)
        except Exception as exc:
            logging.warning("Ollama discovery failed: %s", exc)
            return DiscoveryResult("Ollama", (), Source.EMPTY, str(exc))


    _OLLAMA_DEFAULT_URL = "http://localhost:11434"


    def _canonical_ollama_url(base_url: str | None) -> str:  # audit-added S2
        return (base_url or _OLLAMA_DEFAULT_URL).rstrip("/")


    def discover_models(
        provider: str,
        *,
        api_key: str | None = None,
        ollama_base_url: str | None = None,
        use_cache: bool = True,
    ) -> DiscoveryResult:
        if provider not in providers.PROVIDER_REGISTRY:
            return DiscoveryResult(provider, (), Source.EMPTY, "unknown provider")

        # Audit-added S2: canonicalize identity ONCE — same canonical form drives
        # both the cache lookup and the underlying network call.
        if provider == "Ollama":
            identity: str | None = _canonical_ollama_url(ollama_base_url)
        elif provider == "OpenAI":
            identity = api_key  # None and explicit None share an entry; different non-None values do not
        else:
            identity = None

        cache_key = (provider, identity)

        if use_cache and cache_key in _CACHE:
            return _CACHE[cache_key]

        if provider == "OpenAI":
            result = _discover_openai(api_key)
        elif provider == "Ollama":
            result = _discover_ollama(identity)  # type: ignore[arg-type]  # identity is canonical str
        else:
            # Kokoro / future local-hf providers don't have a discovery surface in v0.1.
            # They expose a fixed fallback list from the registry.
            cap = providers.PROVIDER_REGISTRY[provider]
            result = DiscoveryResult(provider, tuple(cap.fallback_models), Source.FALLBACK, None)

        # Audit-added S4 — intentional: FALLBACK and EMPTY are cached identically to LIVE.
        # Sticky-until-invalidate semantics; "Refresh Models" button is the recovery path.
        # v0.1 contract: discovery is called from the main thread only (no concurrent writes).
        _CACHE[cache_key] = result
        return result
    ```

    - Touching `providers.py` (boundary)
    - Time-based TTL cache invalidation (out of scope — explicit invalidate only)
    - Returning lists instead of tuples (DiscoveryResult is frozen; tuples are hashable + immutable)
